Movie Reel

Movie enthusiasts want to be able to access information about different movies, directors, and genres. The server-side of the myFlix application will ensure they have access to this information, that their requests can be processed, and that all necessary data can be stored.

{ id: '12283080d8038d08d0d', Username: 'Mia', Password: 'hellopassword', Email: '456@gmail.com', Birthday: '01/01/2000', FavoriteMovies:['389y9d8d9'] }

Business Logic URL HTTP Method Request body data format Response body data format
Return a list of ALL movies to the user /Movies GET None JSON object holding data about all of the movies
Return data about a single movie by title to the user /Movies/:Title GET None JSON object holding data about the movie selected by the user
Return data about a genre by movie title /Genre/:Name GET None JSON object holding data about the genre of the movie that was selected by the user
Return data about a director by name /Director/:Name GET None JSON object holding data about the director selected by the user
Allow new users to register /Users POST A JSON object holding data about the user like so:

{ Username: 'Mia', Password: 'hellopassword', Email: '456@gmail.com', Birthday: '01/01/2000', FavoriteMovies:[] }

A JSON object holding data about the user, with an automatically assigned id:

{ id: '12283080d8038d08d0d', Username: 'Mia', Password: 'hellopassword', Email: '456@gmail.com', Birthday: '01/01/2000', FavoriteMovies:[] }

Allow users to update their user info /users/:username POST JSON object holding existing user information like so (ID cannot be changed and users are not required to change any specific field):

{ Username: 'Mia', Password: 'hellopassword', Email: '456@gmail.com', Birthday: '01/01/2000', FavoriteMovies:[] }

JSON obejct confirming the number of user objects modified and how many fields were changed(if the two fields equal 0 that means the username was not recognized):

{ usersChanged: 1, fieldsChanged: 1 }

Allow users to add a movie to their list of favorites /users/:username/FavoriteMovies/:ObjectId POST None JSON object holding data about the user with the updated version of their movie list:

{ id: '12283080d8038d08d0d', Username: 'Mia', Password: 'hellopassword', Email: '456@gmail.com', Birthday: '01/01/2000', FavoriteMovies:['389y9d8d9'] }

Allow users to remove a movie from their list of favorites /users/:username/FavoriteMovies/:ObjectId DELETE None JSON object with data about the user, showing the list of favorite movies without the deleted movie/movies
Allow existing users to deregister /Users/:Name None Text message confirming that the users account has been deleted:

Mia was successfully deleted
