Movie Reel

Movie enthusiasts want to be able to access information about different movies, directors, and genres. The server-side of the myFlix application will ensure they have access to this information, that their requests can be processed, and that all necessary data can be stored.

Business Logic URL HTTP Method Request body data format Response body data format
Return a list of ALL movies to the user /movies GET None JSON object holding data about all of the movies
Return data about a single movie by title to the user /movies/:Title GET None JSON object holding data about the movie selected by the user
Return data about a genre by movie title /Movies/:Title/genre GET None JSON object holding data about the genre of the movie that was selected by the user
Return data about a director by name /directors/:name GET None JSON object holding data about the director selected by the user
Allow new users to register /users POST A JSON object holding data about the user like so:

{ username: 'Mia', password: 'hellopassword', email: '456@gmail.com', birthdate: '01/01/2000', favoritemovies:[] }

A JSON object holding data about the user, with an automatically assigned id:

{ id: '12283080d8038d08d0d', username: 'Mia', password: 'hellopassword', email: '456@gmail.com', birthdate: '01/01/2000', favoritemovies:[] }

Allow users to update their user info /users/:username POST JSON object holding existing user information like so (ID cannot be changed and users are not required to change any specific field):

{ username: 'Mia', password: 'hellopassword', email: '456@gmail.com', birthdate: '01/01/2000', favoritemovies:[] }

JSON obejct confirming the number of user objects modified and how many fields were changed(if the two fields equal 0 that means the username was not recognized):

{ usersChanged: 1, fieldsChanged: 1 }

Allow users to add a movie to their list of favorites /users/:username/favorites/:movieID POST None JSON object holding data about the user with the updated version of their movie list:

{ id: '12283080d8038d08d0d', username: 'Mia', password: 'hellopassword', email: '456@gmail.com', birthdate: '01/01/2000', favoritemovies:['389y9d8d9'] }

Allow users to remove a movie from their list of favorites /users/:username/favorites/:movieID DELETE None JSON object with data about the user, showing the list of favorite movies without the deleted movie/movies
Allow existing users to deregister /users/:username None Text message confirming that the users account has been deleted:

Mias account has been deleted
